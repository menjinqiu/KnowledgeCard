# KnowledgeCard 打印视觉系统重构 Phase A

生成时间：2026-06-26

## 目标

本阶段只优化打印页的视觉体验、纸面排版、色彩规范和多场景适配，让 KnowledgeCard 打印输出从“能打印”升级为“像精心设计过的学习资料”。

核心体验目标：

```text
安静、克制、精致、有留白、有节奏、低饱和、高可读性、纸面友好、适合标注、适合复习、适合长期保存。
```

## 明确不处理

```text
不处理 printable 字段
不处理不同卡片类型的独立模板体系
不改 Dexie schema
不改导入导出协议
不改同步内核
不引入 PDF 生成库
不把 npm run build 结果冒充浏览器打印 QA
```

## 视觉方向

打印成品应像：

```text
高质量课程讲义
现代学习手册
设计良好的知识资料
适合长期保存的技术/学习笔记
```

不应像：

```text
网页截图
后台管理页面打印版
PPT 风格彩色块
密密麻麻的资料堆叠
```

## 色彩原则

比例建议：

```text
85% 黑白灰
10% 柔和纸面/浅灰分区
5% 低饱和彩色强调
```

颜色只承担定位、分组、强调和气质营造，不承担正文阅读。

```css
--print-paper: #ffffff;

--print-ink: #151515;
--print-ink-soft: #3f3b36;
--print-muted: #6d6860;

--print-accent: #315f7d;
--print-accent-soft: #e8f0f5;
--print-accent-2: #2f766d;
--print-accent-2-soft: #e8f3f0;
--print-warm: #9a6b2f;
```

允许彩色的位置：

```text
封面细线
章节标题下划线
目录编号
摘要块左边线
copyText 块左边线
重要提示小标签
页眉/页脚极小装饰
```

避免彩色的位置：

```text
正文大段文字
正文大面积背景
整张卡片大色块
代码块背景
大面积标签填充
大面积渐变背景
```

## A4 版式原则

普通打印模式建议：

```css
@page {
  size: A4;
  margin: 15mm 16mm 17mm;
}
```

正文阅读区：

```text
max-width: 150mm–158mm
margin-inline: auto
中文每行约 30–42 字
英文每行约 55–72 字符
```

间距 scale：

```text
2mm  微间距
4mm  小间距
6mm  标准块间距
8mm  大块间距
12mm 页面级间距
```

字号建议：

```text
封面标题：30–34pt
章节标题：18–22pt
卡片标题：18–20pt
正文标题 h2：13–14pt
正文标题 h3：11.5–12pt
正文：10.5–11pt
辅助信息：8.5–9pt
代码 / copyText：9.5–10.5pt
```

行高建议：

```text
中文正文：1.65–1.75
英文正文：1.55–1.65
代码 / 提示词：1.5–1.6
```

## 单页卡片结构

理想纸面结构：

```text
轻量页眉 / Card 01
卡片标题
domain · type · updatedAt
细分隔线
摘要：左侧蓝灰强调线
正文：清晰 Markdown 排版
可复用内容 / copyText：左侧青绿强调线
页脚：KnowledgeCard · 日期
```

标题要像文档标题，不像网页卡片。
摘要应像导读。
正文不应有沉重的整块边框。
copyText 应像工具块，但不能刺眼。
页脚要弱，不抢注意力。

## 多场景适配要求

必须覆盖：

```text
短卡片：不强行塞满，允许高级留白
长卡片：标题/摘要尽量不拆，正文自然分页
长 copyText：保留换行，允许跨页，避免整块强制不分页造成大空白
标签很多：标签弱化，不喧宾夺主
无摘要 / 无 copyText：不显示空标题，正文自然上移
德语/中英混排：行高舒适，避免拥挤
紧凑连续模式：减少间距和字号，但保持清晰
```

## Phase A 执行范围

允许修改：

```text
src/styles/print.css
必要时小幅调整 print 相关 class
CURRENT_STATE.md
NEXT_SESSION_PROMPT.md
```

优先只修改 CSS，不改业务逻辑。

## 验收标准

```text
1. A4 打印预览中，封面像正式资料，不像网页。
2. 目录有清楚层次，编号有轻微设计感。
3. 单张卡片标题区舒展，不拥挤。
4. 摘要和正文区别明显，但不花哨。
5. copyText 明显可识别，但不刺眼。
6. 正文行长不过宽。
7. 正文行距适合阅读。
8. 标签不喧宾夺主。
9. 长文本分页不出现大面积奇怪空白。
10. 黑白打印仍然可读。
11. 彩色打印有轻微高级感。
12. 另存 PDF 后适合长期保存。
```

本阶段可以通过 `npm run build` 做代码层验证，但不能声称浏览器打印 QA 已完成，除非实际打开 Chrome 系统打印预览或另存 PDF 验证。
